# Getting Started

## Prerequisites

- Node.js 22 LTS
- Yarn 1.22.x (the `packageManager` field in `package.json` locks the version)
- The repo cloned from [github.com/opentargets/platform-webapp](https://github.com/opentargets/platform-webapp)

## Install and run

```bash
# From the repo root — installs every workspace
yarn install

# Start this Storybook (http://localhost:6006)
yarn dev:docs
```

`yarn dev:docs` opens the docs site you are reading now. Sections stories can make real GraphQL calls against the production API, so most components render with live data on first load — no extra setup or local backend required. For how that wiring works, see **Architecture / Storybook preview environment**.

To run the full platform app instead:

```bash
yarn dev:platform   # http://localhost:3000
```

## Root scripts

| Script | What it does |
|---|---|
| `yarn dev:docs` | Docs dev server (Storybook, port 6006) |
| `yarn dev:platform` | Platform app dev server |
| `yarn build:docs` | Static Storybook build → `apps/docs/storybook-static/` |
| `yarn build:platform` | Platform production build |
| `yarn lint` | Biome check across all workspaces |
| `yarn check:fix` | Biome format + lint autofix |

## Where things live

- **Component stories** live *in the package they document* (`packages/ui`, `packages/sections`) next to the source — never copied into `apps/docs`.
- **Narrative docs pages** (like this one) are `.mdx` files under `apps/docs/src/docs/`.

Storybook hot-reloads on save, so no config change is needed when you add either.

## Adding a UI component story

```tsx
// packages/ui/src/components/MyComponent/MyComponent.stories.tsx
import type { Meta, StoryObj } from "@storybook/react";
import MyComponent from "./MyComponent";

const meta: Meta<typeof MyComponent> = {
  title: "UI Components/MyComponent",
  component: MyComponent,
  parameters: { layout: "centered" },
  tags: ["autodocs"],   // opt-in: generates a Docs page for this component
};

export default meta;
type Story = StoryObj<typeof MyComponent>;

export const Default: Story = {
  args: { /* realistic props */ },
};
```

## Adding a sections story

Sections are data-fetching widgets. The `OTApolloProvider`, router, and theme decorators are already wired in `.storybook/preview.ts`, so a story only needs its `meta` and args:

```tsx
// packages/sections/src/target/MySection/MySection.stories.tsx
import type { Meta, StoryObj } from "@storybook/react";
import { MySection } from "./MySection";

const meta: Meta<typeof MySection> = {
  title: "Sections/Target/MySection",
  component: MySection,
  tags: ["autodocs"],
};
export default meta;
```

## Adding a narrative docs page

Drop an `.mdx` file under `apps/docs/src/docs/`. The `<Meta title="...">` path sets its position in the sidebar:

```mdx
import { Meta } from "@storybook/addon-docs/blocks";

<Meta title="My Section/My Page" />

# My Page
Content here.
```

## Before you commit

- **Formatting and linting** use Biome (not ESLint/Prettier). Run `yarn check:fix`; a Husky pre-commit hook enforces it.
- **TypeScript** is strict in every package — `noUnusedLocals`, `noUnusedParameters`, and `verbatimModuleSyntax` are all on. Fix types rather than reaching for `// @ts-ignore`.
